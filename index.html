What is nice.lgbt?

nice.lgbt is a project to enable people to host a website without having to pay for a domain or hosting through the use of GitHub pages. Sites can be of static HTML content or be using Jekyll.

How can I have my own site?

It's fairly easy, but you will need some prerequisites:

  • A GitHub account
  • (optional) Have Git installed on your system

Now you need to head over to the nicelgbt/requests repo and open a new issue. This is further explained in the repo's readme.
After some time, a repository will be created for you. This is where your website will live. The content of this repo always reflects the current version of your website. In order to have your website go live, you must push to the gh-pages branch.
Your website will normally live at nice.lgbt/yourreponame.

How can I get a custom subdomain?

You can either include it in your initial issue on the requests repo or you can email me at fen@nice.lgbt. If you have a subdomain created for you, please follow this guide to set your website to be hosted on the subdomain.

How do I edit my website?

There's two ways. If you know Git, good! Just clone your repository and edit it locally with a markdown or code editor, then push it when you're done! The changed should update instantly.
If you don't, that's fine! GitHub provides a web editor that lets you edit your files from the web! You can also upload files, just search for an "Upload Files" button.

Can I really include anything on my website?

Well, almost anything. If we detect a Code of Conduct violation within the content of your website, we will ask you to remove the offending content or, if this has happened before, take down your website.
